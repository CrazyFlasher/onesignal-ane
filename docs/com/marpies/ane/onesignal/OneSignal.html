init()method 
public static function init(oneSignalAppID:String, autoRegister:Boolean = false, showLogs:Boolean = false):Boolean

Initializes extension context and native SDKs. Call as early as possible to be able to retrieve notifications which are launching the application.

Parameters

oneSignalAppID:String — ID of the app created in the OneSignal dashboard.
 
autoRegister:Boolean (default = false) — iOS only - Set to true to register with notification server immediately after initialization. If set to false, OneSignal.register() must be called later to successfully register with Apple servers and receive notifications.
 
showLogs:Boolean (default = false) — Set to true to show extension log messages.

Returns
Boolean — true if the extension context was created, false otherwise

See also

register()
register()method 
public static function register():void

(iOS only) - Call this method when you want to prompt the user to accept push notifications. Android devices automatically register silently during initialization.

Extension must be initialized using OneSignal.init() with autoRegister parameter set to false before calling this method.

Consider using OneSignal.sendTags() to send more than one tag at a time.

Extension must be initialized using OneSignal.init() before calling this method.

sendTags()method 
public static function sendTags(tags:Object):void

Tag a user based on an app event of your choosing so later you can create segments in the OneSignal dashboard to target these users.

This method allows sending multiple tags at once. Consider using OneSignal.sendTag() to send a single tag.

Extension must be initialized using OneSignal.init() before calling this method.

Parameters

tags:Object — Key-value pairs of your choosing to create or update, for example: 
         OneSignal.sendTags( {
            profession: "warrior",
            area: "desert"
         } );

setSubscription()method 
public static function setSubscription(value:Boolean):void

You can call this method with false to opt users out of receiving all notifications through OneSignal. You can pass true later to opt users back into notifications. Extension must be initialized using OneSignal.init() before calling this method.

Parameters
